# 5×5 矩阵按键接入方案

## 1. 最终结论

当前工程采用以下方案：

- MCU：STM32F429IGT6。
- RTOS：FreeRTOS，使用现有 `UITask`。
- GUI：LVGL 9.5.0。
- 不新增按键任务，不使用 EXTI 扫描，不使用 FreeRTOS 软件定时器。
- 由 LVGL keypad indev 每 10 ms 调用一次非阻塞矩阵扫描。
- 每次完整扫描 5×5，连续 3 次一致后确认按下或释放，消抖约 30 ms。
- 只支持单键；检测到多键或未知位置时立即释放当前软件键并锁定，连续 3 次全释放后恢复。
- 长按和自动连发交给 LVGL，建议 400 ms 开始长按、100 ms 重复。
- BSP 只处理 GPIO、位置、消抖和稳定键值，不处理页面切换。
- UI 层决定方向键当前应解释为 `LEFT/RIGHT/UP/DOWN`，还是 `PREV/NEXT`。
- 旧版工程只用于恢复键位布局和 UI 语义，不复用旧扫描逻辑。

### 3.2 列输入

| 列 | GPIO | 当前模式 | 当前内部上下拉 |
|---|---|---|---|
| C1 | PB8 | 输入 | 无 |
| C2 | PB6 | 输入 | 无 |
| C3 | PB5 | 输入 | 无 |
| C4 | PB4 | 输入 | 无 |
| C5 | PB3 | 输入 | 无 |

定义：`Core/Inc/main.h:71-80`。

初始化：`Core/Src/gpio.c:78-84`。

用户已确认：**列信号在 PCB 上已有外部上拉**。

因此当前列 GPIO 保持 `GPIO_NOPULL` 是合理的，不需要再启用内部上拉。扫描采用低有效逻辑：

- 无按键：列为高。
- 当前扫描行拉低。
- 按键闭合：对应列读到低。

### 3.3 推荐修改行输出模式

旧版和当前工程都把行配置为推挽输出。旧版扫描时让当前行输出低、其他行输出高；多键按下时可能把推挽高电平行和推挽低电平行通过按键矩阵连通，产生 GPIO 争用。

推荐在 `.ioc` 中把 R1～R5 改为：

- 模式：`GPIO_MODE_OUTPUT_OD`，开漏输出。
- 默认电平：高。
- Pull：`GPIO_NOPULL`。
- Speed：Low 或 Medium。

列已有外部上拉，开漏行写高时进入高阻态，当前扫描行写低时有效。这样即使误按多键，也不会形成两个推挽输出直接争用。

如果行必须保持推挽输出，则非当前行应动态切换为输入高阻态，不能简单输出高电平。该实现更复杂，不推荐。

### 3.4 二极管

当前无法从代码确认矩阵是否带逐键二极管。本方案不依赖二极管实现组合键：

- 只接受一个有效按键。
- 两个或更多交点一律进入多键锁定。
- 不从鬼键结果中猜测用户意图。

## 5. 旧版扫描逻辑不能复用的原因

旧版 `key_scan()` 有以下问题：

1. 只定义并扫描 4 行，R5 没有进入扫描数组。
2. R5 初始化为低且不会在扫描时恢复高，若 R5 实际接键可能干扰同列扫描。
3. 每扫描一行调用 `HAL_Delay(10)`。
4. 检测按下后调用 `HAL_Delay(20)` 消抖。
5. 确认按下后忙等到松键，`key_scan()` 才返回。
6. 按住期间会阻塞 `lv_timer_handler()`，影响整个 LVGL 调度。
7. LVGL 无法持续得到 `PRESSED`，长按和自动连发不能正常工作。
8. 只返回扫描顺序中的第一个键，没有多键或鬼键检测。
9. RETURN 在 indev read callback 中直接调用 `ReturnNavBar()`，输入层与页面强耦合。
10. 测量中任意键停止业务也直接写在 read callback 中。
11. 旧版修改 LVGL `lv_group.h` 增加 `KEY_0`～`KEY_9`，不应复制到当前第三方库。

旧版代码只用于恢复布局和产品行为，不作为当前实现模板。

## 6. 推荐分层

```text
GPIO 矩阵
    ↓
Core/BSP/keypad
    完整扫描、消抖、多键锁定、稳定 KEY_CODE
    ↓
Middlewares/lvgl/porting/lv_port_keypad
    LVGL indev、10 ms timer、键值转换、长按参数
    ↓
UI 输入策略
    决定方向键映射为方向事件或 PREV/NEXT
    ↓
lv_group / 焦点对象 / LV_EVENT_KEY
```

### 6.1 BSP 层

使用现有文件：

```text
Core/BSP/Inc/keypad.h
Core/BSP/Src/keypad.c
```

负责：

- 5 行和 5 列 GPIO 表。
- 位置到 `KEY_CODE` 的布局表。
- 一次完整 5×5 扫描。
- 10 ms × 3 次消抖。
- 多键和未知位置锁定。
- 返回稳定按下状态。

不得：

- 包含 `lvgl.h`。
- 调用页面切换函数。
- 调用 `HAL_Delay()` 或 `osDelay()`。
- 忙等松键。
- 生成长按重复事件。

### 6.2 LVGL 适配层

建议新增：

```text
Middlewares/lvgl/porting/lv_port_keypad.h
Middlewares/lvgl/porting/lv_port_keypad.c
```

负责：

- 创建 `LV_INDEV_TYPE_KEYPAD`。
- 注册 read callback。
- 把 indev 读取周期设置为 10 ms。
- 设置 400 ms 长按和 100 ms 重复周期。
- 把 `KEY_CODE` 映射为 LVGL 键值或 ASCII。
- 允许 UI 层注册键值映射策略。

### 6.3 UI 输入策略

负责：

- 普通焦点导航时，把 LEFT/UP 映射为 `LV_KEY_PREV`，RIGHT/DOWN 映射为 `LV_KEY_NEXT`。
- 表格翻页、光标移动、下拉框展开等场景，保留 `LV_KEY_LEFT/RIGHT/UP/DOWN`。
- RETURN 映射为返回事件，不直接在 BSP 或 read callback 中切页面。
- 测量中任意键停止等业务行为放在 UI/业务状态层。

## 7. 扫描和消抖设计

### 7.1 扫描周期

- LVGL indev timer：10 ms。
- 每次 callback 完整扫描 5 行。
- 单次完整扫描目标耗时小于 100 μs。
- 不使用毫秒延时。

### 7.2 扫描电平

前提：列有外部上拉，行是开漏输出。

1. 所有行写高，进入高阻态。
2. 当前行写低。
3. 读取 C1～C5。
4. 当前行写高。
5. 扫描下一行。
6. 扫描结束后统计全部低电平交点。

原始结果分为：

```text
RAW_NONE       没有按键
RAW_SINGLE     只有一个已映射按键
RAW_INVALID    多键，或按下了尚未映射的位置
```

### 7.3 消抖状态

- 按下：同一个有效键连续出现 3 次才确认。
- 释放：连续 3 次无键才确认。
- 按住：持续返回同一键和 `PRESSED`。
- A 按住时检测到 B：立即释放 A，进入锁定。
- 多键：立即释放当前键，进入锁定。
- 未映射位置：进入锁定，避免误操作。
- 锁定解除：连续 3 次全释放。

## 8. LVGL 9.5 keypad 键值行为

当前 LVGL 9.5 的 keypad 路径位于：

- `Middlewares/lvgl/src/indev/lv_indev.c:836-923`

正确行为是：

| 键值 | LVGL keypad 行为 |
|---|---|
| `LV_KEY_NEXT` | 调用 `lv_group_focus_next()` |
| `LV_KEY_PREV` | 调用 `lv_group_focus_prev()` |
| `LV_KEY_ENTER` | 给焦点对象发送 KEY、PRESSED，释放时产生 CLICKED |
| `LV_KEY_ESC` | 给焦点对象发送 KEY 和 CANCEL |
| `LV_KEY_LEFT/RIGHT/UP/DOWN` | 通过 `lv_group_send_data()` 形成焦点对象的 `LV_EVENT_KEY` |
| ASCII `'0'`～`'9'` | 作为普通字符键发送给焦点对象 |

需要特别区分：

- keypad 的焦点切换键是 `LV_KEY_PREV/NEXT`。
- `LV_KEY_LEFT/RIGHT` 不会在 keypad 路径中自动切焦点，而是送给当前焦点对象。
- encoder 路径才有 LEFT/RIGHT 转编码器步进的处理，不能与 keypad 路径混淆。

因此不必为了让控件收到 LEFT/RIGHT 而定义自定义方向键。自定义键只用于 TEST、SIGN 等 LVGL 没有内置语义的按键。

### 8.1 数字键

直接返回 ASCII：

```c
'0' ... '9'
```

不要像旧版一样修改 LVGL 的 `lv_group.h` 添加 `KEY_0`～`KEY_9`。

### 8.2 DELETE

旧版把 `LV_KEY_DEL` 当作删除光标前字符使用，但 LVGL textarea 的标准语义是：

- `LV_KEY_BACKSPACE`：删除前一个字符。
- `LV_KEY_DEL`：删除后一个字符。

参考代码按旧产品实际行为把 `KEY_DELETE` 映射为 `LV_KEY_BACKSPACE`。如果产品要求 Forward Delete，再改成 `LV_KEY_DEL`。

### 8.3 RETURN

旧版 RETURN 直接调用 `ReturnNavBar()`，不推荐继续使用。

当前建议：

- `KEY_RETURN` 映射为 `LV_KEY_ESC`。
- 焦点对象或页面控制器处理 `LV_EVENT_CANCEL`。
- 页面返回逻辑不进入 BSP 和 indev read callback。

```c
#define YOUJIESUN_KEYPAD_H

#include <stdbool.h>

typedef enum {
    KEY_NONE = 0,
    KEY_UP,
    KEY_DOWN,
    KEY_LEFT,
    KEY_RIGHT,
    KEY_ENTER,
    KEY_RETURN,
    KEY_DELETE,
    KEY_TEST,
    KEY_SIGN,
    KEY_0,
    KEY_1,
    KEY_2,
    KEY_3,
    KEY_4,
    KEY_5,
    KEY_6,
    KEY_7,
    KEY_8,
    KEY_9
} KEY_CODE;
```

### 9.3 `Core/BSP/Src/keypad.c`

```c
#include "keypad.h"
#include "main.h"

#define KEYPAD_ROW_COUNT       5U
#define KEYPAD_COLUMN_COUNT    5U
#define KEYPAD_DEBOUNCE_COUNT  3U

typedef struct {
    GPIO_TypeDef * port;
    uint16_t pin;
} keypad_gpio_t;

typedef enum {
    KEYPAD_RAW_NONE = 0,
    KEYPAD_RAW_SINGLE,
    KEYPAD_RAW_INVALID
} keypad_raw_type_t;

typedef struct {
    keypad_raw_type_t type;
    KEY_CODE key;
} keypad_raw_state_t;

static const keypad_gpio_t keypad_rows[KEYPAD_ROW_COUNT] = {
    {KB_R1_GPIO_Port, KB_R1_Pin},
    {KB_R2_GPIO_Port, KB_R2_Pin},
    {KB_R3_GPIO_Port, KB_R3_Pin},
    {KB_R4_GPIO_Port, KB_R4_Pin},
    {KB_R5_GPIO_Port, KB_R5_Pin}
};

static const keypad_gpio_t keypad_columns[KEYPAD_COLUMN_COUNT] = {
    {KB_C1_GPIO_Port, KB_C1_Pin},
    {KB_C2_GPIO_Port, KB_C2_Pin},
    {KB_C3_GPIO_Port, KB_C3_Pin},
    {KB_C4_GPIO_Port, KB_C4_Pin},
    {KB_C5_GPIO_Port, KB_C5_Pin}
};

static const KEY_CODE keypad_layout[KEYPAD_ROW_COUNT][KEYPAD_COLUMN_COUNT] = {
    {KEY_NONE, KEY_LEFT, KEY_RIGHT, KEY_RETURN, KEY_ENTER},
    {KEY_1, KEY_2, KEY_3, KEY_DELETE, KEY_UP},
    {KEY_4, KEY_5, KEY_6, KEY_0, KEY_DOWN},
    {KEY_7, KEY_8, KEY_9, KEY_NONE, KEY_NONE},
    {KEY_NONE, KEY_NONE, KEY_NONE, KEY_NONE, KEY_NONE}
};

static keypad_state_t stable_state;
static KEY_CODE press_candidate;
static uint8_t press_count;
static uint8_t release_count;
static uint8_t unlock_count;

static void keypad_set_all_rows_inactive(void)
{
    for (uint32_t row = 0; row < KEYPAD_ROW_COUNT; row++) {
        HAL_GPIO_WritePin(keypad_rows[row].port, keypad_rows[row].pin, GPIO_PIN_SET);
    }
}

static keypad_raw_state_t keypad_scan_raw(void)
{
    keypad_raw_state_t result = {
        .type = KEYPAD_RAW_NONE,
        .key = KEY_NONE
    };
    uint32_t pressed_count = 0;
    uint32_t pressed_row = 0;
    uint32_t pressed_column = 0;

    keypad_set_all_rows_inactive();

    for (uint32_t row = 0; row < KEYPAD_ROW_COUNT; row++) {
        HAL_GPIO_WritePin(keypad_rows[row].port, keypad_rows[row].pin, GPIO_PIN_RESET);
        __DSB();

        for (uint32_t column = 0; column < KEYPAD_COLUMN_COUNT; column++) {
            if (HAL_GPIO_ReadPin(keypad_columns[column].port, keypad_columns[column].pin) == GPIO_PIN_RESET) {
                pressed_count++;
                pressed_row = row;
                pressed_column = column;
            }
        }

        HAL_GPIO_WritePin(keypad_rows[row].port, keypad_rows[row].pin, GPIO_PIN_SET);
    }

    if (pressed_count == 0U) {
        return result;
    }

    if (pressed_count > 1U) {
        result.type = KEYPAD_RAW_INVALID;
        return result;
    }

    result.key = keypad_layout[pressed_row][pressed_column];
    result.type = result.key == KEY_NONE ? KEYPAD_RAW_INVALID : KEYPAD_RAW_SINGLE;
    return result;
}

static void keypad_reset_press_filter(void)
{
    press_candidate = KEY_NONE;
    press_count = 0;
}

static void keypad_enter_lock(void)
{
    stable_state.key = KEY_NONE;
    stable_state.pressed = false;
    stable_state.multi_locked = true;
    release_count = 0;
    unlock_count = 0;
    keypad_reset_press_filter();
}

void keypad_init(void)
{
    keypad_set_all_rows_inactive();

    stable_state.key = KEY_NONE;
    stable_state.pressed = false;
    stable_state.multi_locked = false;
    release_count = 0;
    unlock_count = 0;
    keypad_reset_press_filter();
}

void keypad_process(void)
{
    keypad_raw_state_t raw = keypad_scan_raw();

    if (stable_state.multi_locked) {
        if (raw.type == KEYPAD_RAW_NONE) {
            if (++unlock_count >= KEYPAD_DEBOUNCE_COUNT) {
                stable_state.multi_locked = false;
                unlock_count = 0;
            }
        }
        else {
            unlock_count = 0;
        }
        return;
    }

    if (stable_state.pressed) {
        if (raw.type == KEYPAD_RAW_SINGLE && raw.key == stable_state.key) {
            release_count = 0;
            return;
        }

        if (raw.type == KEYPAD_RAW_NONE) {
            if (++release_count >= KEYPAD_DEBOUNCE_COUNT) {
                stable_state.key = KEY_NONE;
                stable_state.pressed = false;
                release_count = 0;
            }
            return;
        }

        keypad_enter_lock();
        return;
    }

    if (raw.type == KEYPAD_RAW_INVALID) {
        keypad_enter_lock();
        return;
    }

    if (raw.type == KEYPAD_RAW_NONE) {
        keypad_reset_press_filter();
        return;
    }

    if (press_candidate == raw.key) {
        press_count++;
    }
    else {
        press_candidate = raw.key;
        press_count = 1;
    }

    if (press_count >= KEYPAD_DEBOUNCE_COUNT) {
        stable_state.key = raw.key;
        stable_state.pressed = true;
        release_count = 0;
        keypad_reset_press_filter();
    }
}

keypad_state_t keypad_get_state(void)
{
    return stable_state;
}
```

说明：

- `keypad_layout` 只填入旧版已确认的 17 个键。
- TEST、SIGN 和其他未知位置暂时为 `KEY_NONE`。
- 按下未知位置会进入锁定，不会静默映射成其他键。
- 代码扫描全部 R1～R5，不继承旧版只扫描 4 行的问题。
- `__DSB()` 用于保证行 GPIO 写入先于列读取，不是毫秒延时。

### 9.5 `Middlewares/lvgl/porting/lv_port_keypad.c`

```c
#include "lv_port_keypad.h"

#define KEYPAD_READ_PERIOD_MS    10U
#define KEYPAD_LONG_PRESS_MS     400U
#define KEYPAD_REPEAT_PERIOD_MS  100U

static lv_port_keypad_mapper_t key_mapper;
static uint32_t last_lv_key;
static bool last_pressed;

uint32_t lv_port_keypad_default_map(KEY_CODE key)
{
    if (key >= KEY_0 && key <= KEY_9) {
        return (uint32_t)('0' + key - KEY_0);
    }

    switch (key) {
        case KEY_UP:
            return LV_KEY_UP;
        case KEY_DOWN:
            return LV_KEY_DOWN;
        case KEY_LEFT:
            return LV_KEY_LEFT;
        case KEY_RIGHT:
            return LV_KEY_RIGHT;
        case KEY_ENTER:
            return LV_KEY_ENTER;
        case KEY_RETURN:
            return LV_KEY_ESC;
        case KEY_DELETE:
            return LV_KEY_BACKSPACE;
        case KEY_TEST:
            return LV_KEY_APP_TEST;
        case KEY_SIGN:
            return LV_KEY_APP_SIGN;
        default:
            return 0;
    }
}

void lv_port_keypad_set_mapper(lv_port_keypad_mapper_t mapper)
{
    key_mapper = mapper == NULL ? lv_port_keypad_default_map : mapper;
}

static void lv_port_keypad_read(lv_indev_t * indev, lv_indev_data_t * data)
{
    (void)indev;

    keypad_process();
    keypad_state_t state = keypad_get_state();

    if (state.pressed) {
        if (!last_pressed) {
            last_lv_key = key_mapper(state.key);
        }

        data->key = last_lv_key;
        data->state = LV_INDEV_STATE_PRESSED;
    }
    else {
        data->key = last_lv_key;
        data->state = LV_INDEV_STATE_RELEASED;
    }

    last_pressed = state.pressed;
}

lv_indev_t * lv_port_keypad_init(void)
{
    keypad_init();
    key_mapper = lv_port_keypad_default_map;
    last_lv_key = 0;
    last_pressed = false;

    lv_indev_t * indev = lv_indev_create();
    if (indev == NULL) {
        return NULL;
    }

    lv_indev_set_type(indev, LV_INDEV_TYPE_KEYPAD);
    lv_indev_set_read_cb(indev, lv_port_keypad_read);
    lv_indev_set_long_press_time(indev, KEYPAD_LONG_PRESS_MS);
    lv_indev_set_long_press_repeat_time(indev, KEYPAD_REPEAT_PERIOD_MS);

    lv_timer_t * read_timer = lv_indev_get_read_timer(indev);
    if (read_timer != NULL) {
        lv_timer_set_period(read_timer, KEYPAD_READ_PERIOD_MS);
    }

    return indev;
}
```

映射函数只在稳定按下的第一帧执行；按住期间保持同一个 LVGL 键值，避免 UI 模式变化导致一次长按中途换键。

### 9.6 UI 输入策略示例

旧版在 read callback 中直接读取 `CurrentPage`、`NavBarIsFocused` 和 `KeyBoardIsFocused`。当前建议把模式状态放在 UI 输入模块，通过 mapper 注入，不让 LVGL porting 层依赖页面全局变量。

```c
#include "lv_port_keypad.h"

typedef enum {
    UI_INPUT_MODE_FOCUS = 0,
    UI_INPUT_MODE_DIRECTION
} ui_input_mode_t;

static ui_input_mode_t input_mode = UI_INPUT_MODE_FOCUS;

void ui_input_set_mode(ui_input_mode_t mode)
{
    input_mode = mode;
}

static uint32_t ui_keypad_map(KEY_CODE key)
{
    if (input_mode == UI_INPUT_MODE_FOCUS) {
        if (key == KEY_LEFT || key == KEY_UP) {
            return LV_KEY_PREV;
        }

        if (key == KEY_RIGHT || key == KEY_DOWN) {
            return LV_KEY_NEXT;
        }
    }

    return lv_port_keypad_default_map(key);
}

void ui_input_init(void)
{
    lv_port_keypad_set_mapper(ui_keypad_map);
}
```

使用方式：

- 导航栏、按钮组：`UI_INPUT_MODE_FOCUS`。
- 表格翻页、文本光标、展开的下拉框：`UI_INPUT_MODE_DIRECTION`。
- 页面或焦点状态改变时，由 UI 层调用 `ui_input_set_mode()`。
- 不在 `lv_port_keypad.c` 中判断当前页面。

### 9.7 焦点对象接收方向键

在方向模式下，焦点对象可使用 LVGL 9.5 专用 API读取键值：

```c
static void table_key_event_cb(lv_event_t * event)
{
    uint32_t key = lv_event_get_key(event);

    if (key == LV_KEY_LEFT) {
        table_show_previous_page();
    }
    else if (key == LV_KEY_RIGHT) {
        table_show_next_page();
    }
}
```

不要直接解引用 `lv_event_get_param()`，`lv_event_get_key()` 更明确。

### 9.9 `UITask` 接入顺序

```c
void StartUITask(void * argument)
```
